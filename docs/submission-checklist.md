# Submission Checklist

## Required Deliverables

- [x] GitHub repository with clean code
- [x] Project demonstrates memory-driven agent behavior
- [x] Demo-ready web interface
- [x] Explanation of how memory is used
- [x] Technical article draft (`article.md`)
- [x] LinkedIn + video content prepared locally (not committed to public repo)
- [x] Project documentary (`docs/project-documentary.md`)

## Judging Alignment

- Innovation (30%): no-memory vs memory side-by-side + measurable boost
- Use of memory (25%): retain/recall/feedback loop is core product behavior
- Technical quality (20%): modular backend, adapter architecture, tests, eval script
- User experience (15%): focused workflow with high-clarity outputs
- Real-world impact (10%): incident response speed and consistency

## Live Demo Flow

1. Start app with `python -m flashback_ops`
2. Click `Load Demo Memory`
3. Submit a `payments-api` incident
4. Show memory boost and recalled incidents
5. Submit feedback
6. Re-run incident query and show adaptation
